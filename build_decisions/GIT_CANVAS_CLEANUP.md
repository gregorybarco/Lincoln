# Removing Canvas Files Already Tracked by Git

Your `.gitignore` now excludes new canvas files, but old ones already committed
are still being tracked. Run these commands once from your repo root
(`B:\Homebrewed_AI\Lincoln\`) in Git Bash or PowerShell.

---

## Step 1 — Identify what's tracked

```bash
git ls-files | grep -E "\.(py|f90|js|html|css|md|txt|jl|r|mpl|maple)$" | grep -i canvas
```

This shows every canvas-generated file Git is currently tracking.

---

## Step 2 — Remove from tracking WITHOUT deleting from disk

```bash
# Remove all .py files in a canvas/ or canvas_output/ folder
git rm --cached canvas_output/*.py 2>/dev/null || true
git rm --cached canvas_output/*.f90 2>/dev/null || true
git rm --cached canvas_output/*.js 2>/dev/null || true
git rm --cached canvas_output/*.html 2>/dev/null || true
git rm --cached canvas_output/*.md 2>/dev/null || true
git rm --cached canvas_output/*.jl 2>/dev/null || true
git rm --cached canvas_output/*.r 2>/dev/null || true
git rm --cached canvas_output/*.mpl 2>/dev/null || true
```

Or if the files are scattered, use the recursive form:

```bash
# Remove ALL files from tracking that .gitignore now excludes
git rm --cached -r .
git add .
```

**Warning:** `git rm --cached -r . && git add .` is safe but will stage a
lot of changes at once. It is the cleanest one-shot approach -- Git re-adds
everything except what .gitignore now excludes.

---

## Step 3 — Commit the cleanup

```bash
git commit -m "chore: remove canvas output files from tracking (now gitignored)"
```

---

## Step 4 — Verify

```bash
git status
# Should show nothing staged and no canvas files tracked
git ls-files | grep canvas
# Should return empty
```

---

## If You Want to Remove From History Too (Optional)

The above only stops tracking going forward. The files still exist in old
commits. To scrub them from the full commit history (makes the repo smaller
but rewrites history -- only do this if the repo is not shared yet):

```bash
# Requires git-filter-repo (pip install git-filter-repo)
git filter-repo --path canvas_output/ --invert-paths
```

Or use BFG Repo Cleaner for simplicity. For a private local repo where you
are the only contributor, rewriting history is safe.
